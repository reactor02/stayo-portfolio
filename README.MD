# 🏨 STAYO (숙박 예약 플랫폼)

프레임워크 없이 HTML, CSS, JavaScript만으로 구현한 숙박 예약 웹사이트입니다.
메인·검색·숙소 상세부터 장바구니, 결제, 회원 관리, AI 챗봇 고객센터까지 숙박 예약에
필요한 전 과정을 웹 표준 기술만으로 구현했습니다.

## 📌 프로젝트 정보

- **기간** : 2026.02.28 ~ 2026.03.13
- **인원** : 5명 (팀 프로젝트)
- **담당 역할** : AI 챗봇(멀티턴) / 메인 캐러셀 / 예약 입력값 검증 구현

## 🛠 기술 스택

- **Frontend** : HTML5, CSS3, JavaScript (Vanilla)
- **AI** : Google Gemini API (REST)
- **협업 · 설계** : Git, GitHub, draw.io (스토리보드·WBS)

## ✨ 전체 기능

| 모듈 | 설명 |
|---|---|
| 메인 | 캐러셀 배너, 숙소 검색 (지역 · 날짜 · 인원 선택) |
| 인증 / 회원 | 로그인, 회원가입, 아이디 · 비밀번호 찾기, 마이페이지 |
| 숙소 목록 | 숙소 목록 · 검색 결과 조회 |
| 숙소 상세 | 객실 정보, 편의시설(주차 · 조식 · 수영장 등) 안내 |
| 장바구니 / 결제 | 객실 담기, 합계 · 할인 금액 자동 계산, 결제 페이지 |
| 위시리스트 | 지도 기반 찜 목록 |
| AI 고객센터 | Gemini API 연동 멀티턴 챗봇 (회원 / 비회원) |
| 문의 | 1:1 문의 페이지 |

화면은 메인부터 결제까지 총 14개 페이지로 구성했습니다.

## 🙋 담당 기능 상세

### AI 챗봇 (멀티턴 대화)

- fetch로 Gemini REST API를 직접 호출하는 고객센터 챗봇을 구현했습니다.
- 사용자와 모델의 발화를 contents 배열에 누적해 매 요청마다 전체 대화 이력을 전송하는
  방식으로, 이전 대화의 맥락을 기억하는 멀티턴 대화를 구현했습니다.
- 응답 대기 중에는 전송 버튼을 비활성화해 중복 요청을 방지했습니다.
- response.ok 분기와 try/catch로 API 오류와 네트워크 오류를 구분해 사용자에게
  상황에 맞는 안내 메시지를 출력합니다.
- 새 말풍선이 추가되면 채팅창을 최하단으로 자동 스크롤해 항상 최신 대화가 보이게 했습니다.

### 메인 캐러셀

- setInterval과 CSS transform(translateX)을 이용해 일정 간격으로 자동 전환되는
  슬라이드 배너를 구현했습니다.
- 마지막 슬라이드에 도달하면 인덱스를 초기화해 끊김 없이 반복되는 무한 루프로 처리했습니다.

### 예약 입력값 검증

- **날짜** : 체크인 · 체크아웃 입력에 min 속성으로 과거 날짜 선택을 차단하고, 체크인을
  선택하면 체크아웃의 최소 날짜를 체크인 날짜로 갱신해 체크아웃이 체크인보다 빠른
  예약을 원천적으로 방지했습니다.
- **인원** : 팝업형 스테퍼(+/−)로 인원을 입력받고, 0명 미만으로 감소하지 않도록
  검증했습니다. 확정 시 선택 인원이 검색창에 반영됩니다.
- 팝업 내부 클릭은 stopPropagation()으로 닫힘을 막고, 바깥 영역을 클릭할 때만
  닫히도록 이벤트 전파를 제어했습니다.

## 🔥 트러블슈팅

### 챗봇이 직전 대화를 기억하지 못하는 문제

- **문제** : 질문-응답은 동작하지만 "방금 말한 거 다시 알려줘" 같은 이어지는 질문에는
  챗봇이 맥락을 잃고 엉뚱한 답변을 했습니다.
- **해결** : REST API는 요청 간 상태를 저장하지 않는다는 것(stateless)을 파악하고,
  클라이언트에서 사용자 · 모델의 발화를 배열에 누적해 매 요청마다 전체 대화 이력을
  함께 전송하는 구조로 변경했습니다.
- **배운 점** : HTTP 통신의 무상태 특성과, 대화형 AI 서비스에서 컨텍스트를 클라이언트가
  직접 관리해야 하는 이유를 이해하게 되었습니다.

### 인원 선택 팝업이 내부 클릭에도 닫히는 문제

- **문제** : 팝업 바깥을 클릭하면 닫히도록 document에 클릭 이벤트를 걸었더니,
  팝업 내부의 +/− 버튼을 눌러도 팝업이 닫혀버렸습니다.
- **해결** : 내부 클릭이 이벤트 버블링을 타고 document까지 전파되는 것이 원인임을
  파악하고, 팝업 내부 클릭에 stopPropagation()을 적용해 해결했습니다.
- **배운 점** : 이벤트 버블링의 전파 구조와 target / currentTarget의 차이를 실제 버그를
  통해 체득했습니다.

## 💡 배운 점

- 캐러셀을 "슬라이드 한 장 이동 → 자동 반복 → 무한 루프" 순으로 완성해 가며, 기능을
  한꺼번에 만들지 않고 최소 단위부터 점진적으로 구현하는 개발 방식을 익혔습니다.
- 외부 AI API를 직접 연동하면서 fetch, async/await 기반의 비동기 처리와 REST API
  요청 · 응답 구조를 익혔습니다.
- div 남용 대신 main, section 등 시맨틱(semantic) 태그 중심으로 HTML을 작성하며
  의미 중심의 마크업을 배웠습니다.
